# Agnosia: руководство пользователя

> Agnosia помогает держать выбранные Android-приложения отдельно от основной части телефона.  
> Для этого используется **рабочий профиль Android** - отдельное пространство внутри системы со своими приложениями и настройками.

Идея простая: если приложению не нужно постоянно быть рядом с вашими личными данными, его можно перенести в рабочий профиль, запускать только когда нужно, а после использования снова скрывать.

---

## ⚙️ Как это работает

При первом запуске Agnosia предлагает создать рабочий профиль. Это стандартная возможность Android. Внутри такого профиля приложения отделены от личных приложений, личных аккаунтов и части данных основного профиля.

После настройки вы попадаете на экран **"Приложения"**. Там можно скопировать приложение в рабочий профиль, открыть его, скрыть, удалить из рабочего профиля или настроить для него дополнительные ограничения.

### Обычный сценарий

| Шаг | Действие |
| --- | --- |
| 1 | Вы выбираете приложение в личном профиле |
| 2 | Agnosia копирует его в рабочий профиль |
| 3 | Agnsoia создает ярлык для запуска |
| 4 | При запуске через ярлык приложение запускается как обычно |
| 5 | Когда вы перестаёте им пользоваться или блокируете экран, Agnosia снова скрывает приложение |

Скрытое приложение не отображается как обычное приложение рабочего профиля, и Android ограничивает его запуск и работу в фоне.

---

## 🚀 Первый запуск

Во время первичной настройки Agnosia:

- проверит наличие рабочего профиля или создаст его
- попросит нужные системные разрешения
- откроет нужные разделы настроек Android
- после завершения покажет основной экран со списком приложений

Некоторые разрешения Android выдаёт только через системные экраны настроек. Это нормально: Agnosia откроет нужный раздел, а вам останется включить нужный пункт и вернуться назад.

### Зачем нужны разрешения

| Разрешение | Для чего нужно |
| --- | --- |
| **Рабочий профиль** | Создать отдельное пространство и управлять приложениями внутри него |
| **Статистика использования** | Понять, когда изолированное приложение уже не используется и его можно скрыть |
| **Уведомления** | Показывать служебные уведомления, когда это требуется Android |
| **Управление VPN** | Отключать активный VPN перед запуском приложения из рабочего профиля |
| **Установка APK** | Копировать приложения в рабочий профиль |
| **Поверх других окон** | Позволяет Agnosia включать VPN (ограничение Android) |

## 📱 Действия с приложениями

В карточке приложения доступны только те действия, которые подходят для текущего приложения и профиля.

**Личный профиль**

| Действие | Что означает |
| --- | --- |
| **Копировать в рабочий** | Установить копию приложения в рабочий профиль |
| **Переместить в рабочий** | Скопировать приложение в рабочий профиль и удалить его из личного профиля |
| **Удалить** | Удалить приложение из рабочего профиля. Для некоторых системных приложений Android может разрешить только скрытие |
| **Разрешения приложения** | Открывает список разрешений приложения |

**Рабочий профиль**

| Действие | Что означает |
| --- | --- |
| **Копировать в личный** | Скопировать приложение из рабочего профиля обратно в личный профиль |
| **Создать ярлык** | Добавить на рабочий стол ярлык для запуска замороженного приложения |
| **Заморозить принудительно** | Принудительно заморозить приложение если оно активно. |
| **Удалить** | Удалить приложение из рабочего профиля. Для некоторых системных приложений Android может разрешить только скрытие |
| **Изоляция Agnosia** | Включить или выключить управление приложением через Agnosia |
| **Разрешить обмен / Обмен включён** | Включить или отключить обмен между личным и рабочим профилем для этого приложения, например файлы или контакты |
| **Отозвать runtime-разрешения** | Забрать уже выданные разрешения, например доступ к камере, микрофону, геолокации или файлам |

> **Про обмен между профилями:** он нужен не всем приложениям.  
> Если обмен включён, приложение может пользоваться штатными механизмами Android для связи между личным и рабочим профилем. Если выключен, приложение сильнее отделено от личного пространства.

> **Про изоляцию Agnosia:**
> Если опция активна, то Agnosia управляет жизненным циклом приложения и его изоляцией поверх Android.
> Если опция выключена, то приложение уже управляет Android, в таком случае приложения будет ограничено только политиками рабочего профиля Android и при его запуске не будет отключаться VPN, приложение не будет автоматически закрываться после использовния, и не будет автоматически запускаться VPN. Само приложение будет работать как и в личном профиле, но видеть только то что есть в рабочем профиле.
> Если просто - не отключайте изоляцию если не знаете что делаете :D

### Ярлыки скрытых приложений

Так как Agnosia изолирует приложения надежней чем Shelter или Island, из за этого ярылк приложения пропадает с экрана вашего смартфона и Agnosia создает свой. Это технчиеский момент и для вас он ничего не меняет, просто запускайте приложение с ярлыка как обычно.

Когда вы нажимаете такой ярлык:

1. Agnosia "размораживает" приложение.
2. Запускает его.
3. Следит за его работой.
4. Когда приложение больше не используется, "замораживает" его.

Если вы заблокируете экран, Agnosia старается заморозить все приложения рабочего профиля сразу сразу. Это помогает не оставлять изолированные приложения активными после использования.

### Разрешения и риски

Agnosia показывает разрешения приложения и отмечает потенциально опасные сочетания.

Например, приложение с доступом к геолокации, работе в фоне и интернету может быть более рискованным, чем приложение только с доступом к интернету.

> **Подсказка, а не приговор:** оценка риска помогает понять, на что стоит обратить внимание. Окончательное решение всё равно остаётся за вами.

Если приложение находится в рабочем профиле и у него есть уже выданные разрешения, их можно отозвать прямо из Agnosia нажатием кнопки внизу списка разрешений.

## 🔐 VPN и рабочий профиль

Некоторые приложения пытаются понять, используется ли VPN. Если перед запуском приложения из рабочего профиля VPN нужно временно отключить, Agnosia может сделать это автоматически, а после завершения снова включить выбранный VPN-клиент.

### Настройки VPN

| Настройка | Что делает |
| --- | --- |
| **Отключать VPN** | Перед запуском приложения из рабочего профиля Agnosia временно  активный VPN. |
| **Включать VPN после заморозки** | После скрытия приложения Agnosia отправляет команду запуска выбранному VPN-клиенту. |

Когда VPN временно отключён, Agnosia показывает небольшой полупрозрачный **синий квадрат** в правом верхнем углу экрана. Это связано с ограничениями Android и не влияет ни на что.

### Поддерживаемые VPN-клиенты

**FlClash**
**Clash Meta**
**Happ**
**Tunguska**
**INCY**
**Exclave**
**husi**
**NekoBox+**

Для Tunguska нужно дополнительно указать токен автоматизации. Без него Agnosia не сможет отправить команду запуска.

## ⚙️ Настройки приложения

### Тема

| Тема | Как выглядит |
| --- | --- |
| **Красная** | Фирменная тёмная тема Agnosia с красным акцентом. |
| **Темная** | Спокойная тёмная тема. |
| **Светлая** | Светлая тема. |

Тема применяется сразу и сохраняется для следующих запусков.

### Поведение

| Настройка | Простое объяснение |
| --- | --- |
| **Показывать все пакеты** | Показывает не только обычные приложения, но и системные или служебные пакеты |
| **Включить логирование** | Записывает последние события работы Agnosia во внутренний журнал |
| **Отключать VPN** | Временно отключает активный VPN перед запуском приложения из рабочего профиля (Отключает любой VPN) |
| **Включать VPN после заморозки** | После скрытия приложения запускает выбранный VPN-клиент (только поддерживаемые приложения) |

## 📝 Журнал событий

Журнал событий - это короткая история последних действий и ошибок Agnosia. Он нужен для диагностики.

Для того чтобы открыть его нужно нажать на название приложение в верхней части экрана, если у вас активна опция **Включить логирование**, то в жрунале можно будет увидеть диагностические данные, скопировать их или очистить журнал.
Диагностические данные полезно прикладывать в отчетах об ошибке, для этого просто скопируйте их.
Журнал событий может незначительно снижать производительность на слабых устройствах, поэтому лучше не держать его все время включеным
